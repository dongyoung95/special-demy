# 특수데미 사이트 요구사항 및 기획안

## 1. 프로젝트 개요

특수데미는 UDT, SEAL 등 특수부대 선발을 목표로 하는 지원자에게 체력, 수영, 멘탈, 선발 대비 루틴을 교육하는 전문 학원이다.

사이트의 목적은 다음과 같다.

- 특수데미의 전문성과 훈련 철학을 강하게 전달한다.
- 방문자가 오래 머물며 "내가 준비가 부족한가?"를 스스로 점검하게 만든다.
- 블로그, 인스타그램, 유튜브 등 외부 콘텐츠 채널로 자연스럽게 이동시킨다.
- Gmail, 카카오톡 채널, 사이트 문의 양식 중 원하는 방식으로 문의할 수 있게 한다.
- 관리자가 관리자 페이지에서 사이트 문의를 확인하고 답변 상태를 관리할 수 있게 한다.

주의할 점은 특수부대 지원 대비 학원이라는 정체성을 명확히 하되, 무기, 전술, 폭력 행위 교육처럼 오해될 수 있는 표현은 피한다. 사이트 문구는 "군사 전술 교육"이 아니라 "체력, 수영, 정신력, 루틴, 선발 대비 코칭" 중심으로 구성한다.

## 2. 벤치마킹 사이트 분석

참고 사이트: https://isanghan.co.kr/

벤치마킹할 핵심 요소는 다음과 같다.

- "나가도 좋습니다", "문의하지 마세요"처럼 역설적인 문구로 체류시간을 늘린다.
- 주장만 하지 않고 숫자, 성과, 사례, 외부 링크로 신뢰를 만든다.

- 마지막에는 무조건 문의를 강요하기보다 "충분히 읽고 신중히 판단하라"는 분위기로 전환율을 높인다.

- 훈련 난이도와 현실성을 숨기지 않는 톤을 사용한다.
- 합격 사례, 훈련 후기, 훈련 전후 변화, 영상 콘텐츠를 신뢰 근거로 배치한다.
- 문의 전 자가진단과 FAQ를 읽게 만들어 진성 문의 비율을 높인다.

## 3. 추천 사이트 메뉴

상단 메뉴는 다음 구성을 추천한다.

- 홈
- 특수데미 소개
- 훈련 프로그램

- FAQ
- 문의하기

관리자 메뉴는 일반 사용자에게 노출하지 않고 `/admin` 경로로 접근한다.

### 3.1 홈

방문자가 처음 보는 랜딩 페이지다. 어그로성 문구, 신뢰 근거, 프로그램 요약, 후기, 콘텐츠 링크, 문의 유도를 한 화면 흐름으로 구성한다.

### 3.2 특수데미 소개

특수데미의 철학, 운영 방식, 지도자 소개, 안전 원칙, 훈련 대상자를 설명한다.

### 3.3 훈련 프로그램

프로그램은 다음처럼 나눈다.

- UDT/특수부대 선발 대비반
- 수영 및 입영 대비반
- 체력 기초반
- 단기 집중반
- 1:1 상담 및 맞춤 루틴반

각 프로그램에는 대상, 목표, 훈련 내용, 추천 대상, 준비물, 상담 유도 버튼을 포함한다.

### 3.4 합격 사례 및 후기

합격자 인터뷰, 훈련 전후 변화, 수강생 후기, 체력 기록 변화 등을 보여준다. 실제 사례가 부족한 초기에는 "훈련 후기"와 "변화 사례" 중심으로 구성하고, 추후 합격 사례가 생기면 별도 섹션으로 강화한다.

### 3.5 콘텐츠

블로그, 인스타그램, 유튜브로 이동하는 링크를 배치한다.

- 블로그: 긴 글, 합격 준비법, 체력 루틴, 수영 팁
- 인스타그램: 짧은 훈련 영상, 현장 분위기, 공지
- 유튜브: 훈련 설명, 인터뷰, 자가진단 영상

### 3.6 FAQ

반복 문의를 줄이기 위해 다음 질문을 포함한다.

- 체력이 부족해도 시작할 수 있나요?
- 수영을 못해도 등록할 수 있나요?
- 몇 개월 정도 준비해야 하나요?
- UDT, SSU, 해병수색대, 특전사 준비가 모두 가능한가요?
- 훈련 강도는 어느 정도인가요?
- 부상 위험은 어떻게 관리하나요?
- 상담 후 바로 등록해야 하나요?

### 3.7 문의하기

문의 방식은 세 가지로 제공한다.

- Gmail 문의: 버튼 클릭 시 Gmail 작성 또는 `mailto:` 링크로 이동
- 카카오톡 문의: 특수데미 카카오톡 채널 또는 친구추천 페이지로 이동

> 대부분은 지원서를 쓰기 전에 탈락합니다. 체력 때문이 아니라, 준비를 현실로 착각하기 때문입니다.

설명 포인트:

- 유튜브 영상만 보고 훈련을 안다 착각한다.
- 팔굽혀펴기 개수보다 중요한 것은 매일 반복하는 루틴이다.
- 물 적응, 호흡, 체력 회복, 멘탈 관리는 혼자 하기 어렵다.
- 선발 준비는 "열심히"보다 "정확히"가 중요하다.

### 4.3 특수데미가 다른 이유

핵심 메시지:

- 막연한 군대 로망이 아니라 실제 선발 준비에 필요한 몸과 습관을 만든다.
- 체력, 수영, 멘탈, 생활 루틴을 함께 본다.
- 상담 단계에서 무리한 등록을 유도하지 않고 현재 상태를 먼저 진단한다.
- 부상 방지를 위해 단계별 강도 조절을 한다.

### 4.4 프로그램 요약

카드 형태로 주요 프로그램을 보여준다.

- 특수부대 선발 대비반

- 기초 체력 강화반

### 4.5 자가진단 섹션

방문자가 오래 머물게 하기 위해 체크리스트를 제공한다.

예시 질문:

- 3km 달리기 기록을 알고 있는가?
- 자유형으로 200m 이상 연속 수영이 가능한가?
- 주 4회 이상 꾸준히 운동하고 있는가?
- 훈련 일지를 작성하고 있는가?
- 물속에서 호흡이 무너지지 않는가?
- 체력보다 먼저 생활 패턴을 바꿀 준비가 되어 있는가?

결과 문구:

- 0-2개 해당: 상담 전에 기초반부터 확인하는 것을 추천
- 3-4개 해당: 선발 대비 루틴 설계 필요
- 5개 이상 해당: 단기 목표와 기록 향상 전략 상담 추천

## 5. 문의 기능 요구사항

### 5.1 문의 양식 입력값

문의 양식은 다음 항목을 받는다.

- 이름
- 전화번호

- 희망 과정

- 문의 내용

## 8. 확정 기술 구성 (Java / Spring Boot)

구현은 **Java 17 + Spring Boot 3.2** 기반 단일 웹 애플리케이션으로 진행한다. (2026-05-04 반영)

- 언어: Java 17
- 프레임워크: Spring Boot 3.2 (spring-boot-starter-web)
- 뷰: Thymeleaf (서버 사이드 HTML, 공통 헤더·푸터 프래그먼트)
- 스타일: 정적 CSS (`src/main/resources/static/css/main.css`)
- 영속성: Spring Data JPA + Hibernate
- DB: 로컬 개발은 H2 **파일** 저장소 (`jdbc:h2:file:./data/specialdemy`, 프로젝트 루트 기준 `data/` 폴더). **문의·관리자 계정**은 반드시 RDB에 영속화하는 것이 맞으므로 운영·스테이징에서는 MySQL/MariaDB 등으로 전환한다.
- 운영 DB **DDL**: 저장소 루트의 `특수데미.sql` (MySQL 8 / MariaDB 10.5+ 대상, `CREATE TABLE IF NOT EXISTS`만 사용한 **멱등** 스크립트). 엔티티 `Inquiry`, `AdminUser`의 물리 컬럼명(snake_case)과 일치시켰다.
- 보안: Spring Security (폼 로그인, BCrypt 비밀번호, `ROLE_ADMIN`)
- 빌드: Apache Maven. 로컬에 Maven이 없을 경우 저장소에 포함된 **Maven Wrapper** (`mvnw.cmd`) 사용.

### 8.1 실행 방법

- Windows: 프로젝트 루트에서 `mvnw.cmd spring-boot:run`
- **MySQL 개발 서버** (DB `specialdemy`, 계정 `root` / `root`, 앱 포트 `8080`):  
  `mvnw.cmd spring-boot:run -Dspring-boot.run.profiles=dev`  
  설정 파일: `application-dev.yml` (MySQL이 `localhost:3306`에서 떠 있어야 함. 스키마는 `특수데미.sql` 실행 또는 `ddl-auto: update`로 생성·갱신.)
- **운영 서버**:  
  `mvnw.cmd clean package`로 jar를 만든 뒤, 운영 서버에서 `prod` profile로 실행한다.  
  `java -jar target/special-demy-0.0.1-SNAPSHOT.jar --spring.profiles.active=prod`  
  설정 파일: `application-prod.yml` (DB 접속 정보와 관리자 초기 비밀번호는 환경 변수로 주입. 운영 DB 스키마는 배포 전에 `특수데미.sql`로 생성.)
- 기본 URL: `http://localhost:8080/`
- 관리자 문의 목록: `http://localhost:8080/admin/inquiries` (로그인 필요)
- 관리자 로그인: `http://localhost:8080/admin/login`

#### Cursor 에이전트·자동 재시작

- 코드·설정·템플릿·정적 리소스를 수정한 뒤에는, 사용자가 “서버 다시 켜줘”라고 하기 **전에** 에이전트가 `8080`을 비우고 `mvnw.cmd spring-boot:run`(또는 MySQL 개발 중이면 동일하게 `profiles=dev`)으로 서버를 다시 띄우는 것을 원칙으로 한다.
- 상세 규칙: 저장소 `.cursor/rules/spring-boot-restart-after-changes.mdc`

### 8.2 설정 (`application.yml`)

- `special-demy.external.*`: 블로그, 인스타, 유튜브, 문의용 이메일(`contact-email`), 카카오톡 채널 URL
- `special-demy.admin.username` / `special-demy.admin.initial-password`: **DB에 관리자 계정이 없을 때만** 최초 1회 부트스트랩으로 생성되는 계정 정보. 운영에서는 반드시 강한 비밀번호로 바꾸고, 가능하면 환경 변수로 덮어쓴다.
  - 운영 환경 변수 예: `SPECIAL_DEMY_ADMIN_USERNAME`, `SPECIAL_DEMY_ADMIN_INITIAL_PASSWORD`
- `application-dev.yml`: 로컬 MySQL 개발, Thymeleaf 캐시 해제, 정적 리소스·템플릿을 `src/main/resources`에서 직접 제공.
- `application-prod.yml`: 운영 MySQL/MariaDB 접속 정보 환경 변수화, Thymeleaf·정적 리소스 캐시 사용, Hibernate `ddl-auto: validate`.
  - 필수 운영 환경 변수: `SPRING_DATASOURCE_URL`, `SPRING_DATASOURCE_USERNAME`, `SPRING_DATASOURCE_PASSWORD`, `SPECIAL_DEMY_ADMIN_INITIAL_PASSWORD`

### 8.4 기타 구현 메모

- 문의 폼에 봇용 **허니팟** 필드 `_trap`이 채워지면 DB에 저장하지 않고 동일한 완료 화면으로 리다이렉트한다.
- 문의 완료 페이지(`/contact/thanks`)는 세션 플래그로 직접 URL 접근을 차단한다.
- 외부 링크·메일 주소는 `GlobalModelAdvice`로 모든 공개 페이지에 `links` 모델로 주입한다.

### 8.5 운영 DB 스키마 파일 (`특수데미.sql`)

- **필요성**: 사이트 문의(`inquiries`)와 관리자(`admin_users`)는 서버 재시작 후에도 유지되어야 하므로 RDB 테이블이 필요하다. (로컬은 H2로 빠르게 개발 가능.)
- **파일 위치**: 프로젝트 루트 `특수데미.sql`
- **대상 엔진**: MySQL 8.0+ 또는 MariaDB 10.5+ (InnoDB, `utf8mb4`)
- **멱등 정책**: `CREATE TABLE IF NOT EXISTS`만 사용한다. 같은 DB에 대해 스크립트를 여러 번 실행해도 **기존 테이블·데이터를 삭제하지 않으며**, 이미 존재하면 해당 CREATE는 건너뛴다. 인덱스는 테이블 정의에 포함한다(별도 `CREATE INDEX` 없음).
- **애플리케이션 연동 시**: `spring.datasource`를 MySQL로 바꾸고, JPA `ddl-auto`는 운영에서 `validate` 또는 `none`을 권장한다(스키마는 SQL로 관리).

### 8.6 반응형 UI (모바일)

- 뷰포트 메타 태그 유지, 공통 CSS에서 소폭 화면용 간격·타이포 조정.
- **내비게이션**: 약 `720px` 이하에서 햄버거(체크박스 + 라벨)로 메뉴를 접었다 펼 수 있게 한다. 데스크톱에서는 기존 가로 링크 유지.
- **관리자 문의 목록**: 가로 스크롤 가능한 래퍼(`.table-scroll`)로 좁은 화면에서 테이블이 잘리지 않게 한다.
- **터치**: 주요 버튼·메뉴 링크는 최소 높이 약 `2.75rem` 수준으로 탭하기 쉽게 한다.

### 8.7 톤·스크롤 리빌 (애플 랜딩 뉘앙스)

- **톤**: 배경·서피스 컬러를 한 단계 밝혀 과도하게 무겁지 않게 한다(여전히 다크 모드 계열).
- **스크롤 스크럽(애플식)**: 홈 상단 히어로와 일부 스토리 블록에 `[data-scroll-scrub]` 존을 두고, `scroll-scrub.js`가 스크롤 **진행도**에 따라 `[data-scrub-chars]` 문장을 글자 단위로 나눈 뒤 한 글자씩 블러·이동과 함께 드러낸다. `strong` 등 마크업이 필요한 문단은 `[data-scrub-block]`으로 덩어리 페이드인. 존 높이(`min-height`)를 길게 잡아 **스크롤하는 동안** 연출이 이어지게 한다.
- **스크롤 리빌(보조)**: 그 아래 섹션은 `scroll-reveal` + `sr-el`로 뷰포트 진입 시 순차 페이드인한다.
- **접근성**: `prefers-reduced-motion: reduce`일 때는 전환 없이 즉시 표시한다.
- **적용이 안 보일 때**: Spring Boot는 `src/main/resources` 변경 후 **서버 재시작**이 필요할 수 있다. 브라우저는 **강력 새로고침**(Ctrl+F5). 정적 리소스는 `main.css?…`, `scroll-reveal.js?…` 쿼리로 캐시 무력화하고, `spring.web.resources.cache.period=0s`로 브라우저 캐시를 줄인다. 스크립트가 실패해도 본문이 보이도록 `opacity:0`은 `html.use-scroll-reveal`이 붙은 경우에만 적용한다.

## 12. 보안 및 운영 요구사항

- 관리자 페이지는 반드시 인증 후 접근한다.
- 관리자 비밀번호는 해시 처리한다.

## 15. 최종 방향

특수데미 사이트는 단순한 학원 소개 사이트보다 "선발 준비를 진지하게 생각하게 만드는 랜딩 페이지"에 가깝게 설계하는 것이 좋다. 첫 화면에서는 강한 문구로 시선을 끌고, 중간에서는 현실적인 문제를 짚으며, 마지막에는 Gmail, 카카오톡, 사이트 문의 중 편한 방식으로 행동하게 만든다.

관리자 페이지는 초기에는 문의 확인과 상태 관리에 집중한다. 이후 후기, 콘텐츠, 외부 링크 관리 기능을 붙이면 운영자가 개발자 도움 없이도 사이트를 관리할 수 있다.
